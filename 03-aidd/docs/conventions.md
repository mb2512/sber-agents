# Конвенции для генерации кода code-ассистентом

- Пиши максимально просто: минимум кода, максимум прозрачности.
- Убирай все лишнее: только то, что необходимо для запуска и проверки идеи.
- Одна задача — один простой модуль или функция.
- Избегай избыточных классов и абстракций — только если без них нельзя.
- Не пиши boilerplate и не заранее оптимизируй то, что не востребовано прямо сейчас.
- Всё должно быть понятно без лишних комментариев.
- Соблюдай согласованные технологии и структуру из vision.md.
