# Добавление SSH-ключа на сервер

## Ваш публичный SSH-ключ

```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPOm0n2CNgwl1TTc1i+o4VNDukmmBvVlmip+9GMXzut+ finance-bot-server
```

**Скопируйте весь ключ выше** (включая `ssh-ed25519` в начале).

## Способ 1: Через панель управления провайдера

1. Войдите в панель управления вашего хостинг-провайдера
2. Найдите раздел:
   - "SSH Keys"
   - "Access Keys" 
   - "Security"
   - "Server Access"
   - "SSH Access"
3. Нажмите "Add Key" или "Добавить ключ"
4. Вставьте публичный ключ (весь текст выше)
5. Сохраните

## Способ 2: Через веб-консоль сервера

Если у вас есть доступ к веб-консоли (VNC/консоль) через панель управления:

1. Откройте веб-консоль сервера
2. Войдите в систему (обычно `root` или другой пользователь)
3. Выполните команды:

```bash
# Создайте директорию для SSH ключей
mkdir -p ~/.ssh
chmod 700 ~/.ssh

# Добавьте ваш публичный ключ
echo "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPOm0n2CNgwl1TTc1i+o4VNDukmmBvVlmip+9GMXzut+ finance-bot-server" >> ~/.ssh/authorized_keys

# Установите правильные права
chmod 600 ~/.ssh/authorized_keys
```

## Способ 3: Если нужно добавить для конкретного пользователя

Если вы знаете имя пользователя (например, `user`, `ubuntu`, `admin`):

```bash
# В веб-консоли выполните:
sudo mkdir -p /home/ПОЛЬЗОВАТЕЛЬ/.ssh
sudo chmod 700 /home/ПОЛЬЗОВАТЕЛЬ/.ssh
sudo echo "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPOm0n2CNgwl1TTc1i+o4VNDukmmBvVlmip+9GMXzut+ finance-bot-server" >> /home/ПОЛЬЗОВАТЕЛЬ/.ssh/authorized_keys
sudo chmod 600 /home/ПОЛЬЗОВАТЕЛЬ/.ssh/authorized_keys
sudo chown -R ПОЛЬЗОВАТЕЛЬ:ПОЛЬЗОВАТЕЛЬ /home/ПОЛЬЗОВАТЕЛЬ/.ssh
```

## Проверка подключения

После добавления ключа попробуйте подключиться:

```bash
ssh user@195.209.210.204
# или
ssh root@195.209.210.204
# или
ssh ubuntu@195.209.210.204
```

Должно подключиться без запроса пароля.

## Если не знаете имя пользователя

Попробуйте связаться с провайдером или проверить документацию:
- Какое имя пользователя по умолчанию?
- Как добавить SSH-ключ через панель управления?
- Есть ли веб-консоль для доступа к серверу?

## Альтернатива: Временный доступ по паролю

Если провайдер позволяет временно включить парольную аутентификацию:

1. В панели управления включите "Password Authentication"
2. Подключитесь с паролем: `ssh user@195.209.210.204`
3. Добавьте SSH-ключ (команды выше)
4. Отключите парольную аутентификацию обратно
